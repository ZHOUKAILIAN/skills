# skills

个人自用的 skill 仓库。

这里主要用来沉淀在 AI Coding、Agent 协作和日常开发中可重复使用的技能说明、流程模板和配套素材，目标是把零散经验整理成可复用、可维护、可持续迭代的能力单元。

## 这个仓库放什么

- 可直接调用的 skill
- 针对特定任务的工作流说明
- 配套的脚本、示例和素材
- 个人验证过、愿意长期维护的方法论

## 设计原则

- 一个 skill 只解决一类明确问题
- 触发条件要清楚，避免模糊描述
- 步骤要短、直接、可执行
- 优先沉淀可复用的模板，而不是一次性技巧
- 能配脚本就不要只写概念

## 目录约定

建议每个 skill 独立成目录，入口文件统一为 `SKILL.md`。

```text
.
├── README.md
└── <skill-name>/
    ├── SKILL.md
    ├── assets/
    ├── examples/
    └── scripts/
```

## 一个好的 Skill 应该包含什么

- 适用场景
- 明确的触发描述
- 输入和输出约束
- 可执行的步骤
- 边界条件和不适用情况
- 必要时附带示例或脚本

## 使用目标

这个仓库不是为了展示，而是为了自己长期积累和复用。

核心目标只有三个：

1. 减少重复思考
2. 提高任务执行质量
3. 让常用经验可以快速迁移到不同项目
